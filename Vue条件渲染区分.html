Vue条件渲染区分

2021-05-14
vue
字数统计: 503字   |   阅读时长≈ 2分钟
  1. v-if在初始渲染时判断条件，若为假，则不进行渲染。只有当条件为真时才开始渲染条件块。确保在切换过程中条件块内的事件监听器和子组件被适当的销毁和重建。
  2. v-show无论初始判断条件是真或假，均会进行渲染。即只是切换元素的CSS Property display。
  3. v-show不支持<template>和v-else
<ul>
    <li
        v-for="user in users"
        v-if="user.isActive"
        :key="user.id"

        {{ user.name }}
    </li>
</ul>

//改写后
<ul>
    <li
        v-for="user in activeUsers"
        :key="user.id"

        {{ user.name }}
    </li>
</ul>
computed: {
    activeUsers() {
        return this.users.filter(user => user.isActive)
    }
}

//改写前
<ul>
    <li
        v-for="user in users"
        v-if="shouldShowUsers"
        :key="user.id"
    >
        {{ user.name }}
    </li>
</ul>

//改写后
<ul v-if="shouldShowUsers">
    <li
        v-for="user in users"
        :key="user.id"
    >
        {{ user.name }}
    </li>
</ul>
```

改写后不需要对列表内每个用户都检查是否应该显示，而只检查一次，且不会```v-for```不应该显示的用户。
